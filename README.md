# KS1998 Project

This project contains a Julia implementation of the Krusell-Smith (1998) model, organized in a paper-style / replication-style structure.

## Layout

```text
KS1998/
├── src/
│   └── KSFunctions.jl
├── scripts/
│   ├── config.jl
│   ├── run_ks.jl
│   └── make_figures.jl
├── output/
│   ├── figures/
│   ├── logs/
│   ├── results/
│   └── tables/
├── notebooks/
├── main.jl
├── Project.toml
├── README.md
└── KS1998_Solver.ipynb
```

## File Roles

### `src/KSFunctions.jl`
Core model code.

Contains:
- parameter type definition `KSParams`
- grids and pricing functions
- interpolation helpers
- continuous-choice VFI / PFI household solvers
- simulation
- law-of-motion regression
- outer fixed-point solver
- selected diagnostics and helper plotting functions

This file is meant to contain reusable model logic, not a specific calibration.

### `scripts/config.jl`
Calibration and run configurations.

Currently includes:
- `baseline_config()`
- `fast_config()`

This is the intended place to change parameter values, grid sizes, simulation length, damping, and run tolerances.

### `scripts/run_ks.jl`
Solve the KS model and save results.

This script:
- loads the baseline configuration
- runs the outer fixed-point iteration
- recomputes the final household policy at the stored final coefficient
- saves results to `output/results/ks_result.jls`
- writes a short summary to `output/logs/solve_ks_summary.txt`

It defines:
- `run_ks_pipeline()`

### `scripts/make_figures.jl`
Generate figures from a previously saved result file.

This script:
- reads `output/results/ks_result.jls`
- does **not** re-solve the model
- writes figures to `output/figures/`

It defines:
- `make_figures_pipeline()`

### `main.jl`
One-command entry point.

This script runs:
1. `run_ks_pipeline()`
2. `make_figures_pipeline()`

Use this when you want to solve the model and generate figures in one pass.

### `KS1998_Solver.ipynb`
Original notebook workflow.

This notebook was intentionally left untouched.
The script-based structure was added alongside it, not in place of it.

## How To Run

### 1. Solve + save results + make figures

```julia
julia --startup-file=no main.jl
```

### 2. Solve only

```julia
julia --startup-file=no scripts/run_ks.jl
```

### 3. Make figures from saved results only

```julia
julia --startup-file=no scripts/make_figures.jl
```

## Saved Outputs

### `output/results/ks_result.jls`
Serialized Julia result object containing:
- configuration used
- outer-solver result
- final value function
- final policy function
- save timestamp

### `output/logs/solve_ks_summary.txt`
Short text summary of the final solve.

### `output/figures/`
Saved figures generated by `scripts/make_figures.jl`.

## Workflow Recommendation

For model development:
- keep notebook work in `KS1998_Solver.ipynb`
- move stable code into `src/` and `scripts/`

For reproducible runs:
- change parameters in `scripts/config.jl`
- run `main.jl` or `scripts/run_ks.jl`
- generate figures from saved results using `scripts/make_figures.jl`

## Notes

- The current script structure is designed to separate:
  - model logic
  - calibration
  - solving
  - figure production
- `make_figures.jl` reads saved results instead of re-solving the model.
- The existing notebook remains available for experimentation and exposition.
